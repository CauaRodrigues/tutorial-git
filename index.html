1 - Git

1.1 - O que é Git?

Git é um sistema de controle de versão de arquivos. Através deles podemos desenvolver projetos na qual diversas pessoas podem contribuir simultaneamente no mesmo, editando e criando novos arquivos e permitindo que os mesmos possam existir sem o risco de suas alterações serem sobrescritas. Git permite que um arquivo possa ser editado ao mesmo tempo por pessoas diferentes. Por mais complexo que isso seja, ele tenta manter tudo em ordem para evitar problemas para nós desenvolvedores.

Outro fator importante do git é a possibilidade de criar, a qualquer momento, vários snapshots do seu projeto ou branch. Suponha que o seu projeto seja um site html, e você deseja criar uma nova seção no seu código HTML, mas naquele momento você não deseja que estas alterações estejam disponíveis para mais ninguém, só para você. Isso é, você quer alterar o projeto (incluindo vários arquivos nele), mas ainda não quer que isso seja tratado como “oficial” para outras pessoas, então vc cria um branch (como se fosse uma cópia espelho) e então trabalha apenas nesse branch, até acertar todos os detalhes dele. Após isso, você pode fazer um merge de volta do seu branch até o projeto original.

1.2 - Instalando o Git

O git é um programa de versionamento de código que pode ser instalado neste link: https://git-scm.com/downloads para Windows, Mac, ou para plataformas Linux/Debian, como o Ubuntu. Basta acessar o link, escolher a plataforma e seguir os passos de instalação!
Vale lembrar que se você usa uma VM na nuvem, como o cloud9 ou koding, o git já estará disponível em sua linha de comando.

1.3 - Principais Comandos

git add - Adicione o conteúdo do arquivo ao índice
git branch - Listar, criar ou excluir ramos
git checkout - Alternar ramos ou restaurar arquivos da árvore de trabalho
git clone - Clone um repositório em um novo diretório
git commit - Registre as alterações no repositório
git diff - Mostra mudanças entre commits, commit e árvore de trabalho, etc
git init - Crie um repositório Git vazio ou reinicialize um existente
git pull - Busque e integre com outro repositório ou filial local
git push - Atualizar referências remotas junto com objetos associados
git status - Mostra o estado da árvore de trabalho

2 - Primeiros passos & Criando um repositório

2.1 - Configure seu usuário:

O controle de versão controla quem faz as alterações num projeto. Então você deve configurar seu usuário:

$ git config --global user.name "SeuNome"
$ git config --global user.email "meu.email@gmail.com"
(Substitua com seu Usuário e E-mail da sua conta no GitHub.)

2.2 - Iniciando Repositório:

Para criar um novo repositório, vá até a pasta que vai utilizar no projeto e use o comando "git init", um comando que você usa apenas durante a configuração inicial de um novo repositório local. A execução desse comando cria um novo subdiretório (.git) na pasta de trabalho atual. Essa ação também vai criar uma ramificação (branch) principal.

2.3 - Clonando Repositório:

Se um projeto já foi configurado em um repositório remoto, o comando clonar é a forma mais comum para os usuários obterem uma cópia de desenvolvimento local. Como o "git init", a clonagem é geralmente uma operação única. Depois que um desenvolvedor obtiver uma cópia ativa, todas as operações de controle de versão serão gerenciadas pelo seu repositório local.

$ git clone repo url

"git clone" é o comando usado para criar uma cópia ou clone de repositórios remotos. Você transmite o git clone em uma URL de repositório. O Git é compatível com alguns protocolos de rede diferentes e formatos correspondentes de URL. Neste exemplo, usaremos o protocolo SSH de Git. As URLs do SSH de Git seguem um modelo de:

$ git @HOSTNAME: NOME_DE_USUARIO/NOME_DO_REPOSITORIO.git

Quando executado, a versão mais recente dos arquivos do repositório remoto na ramificação principal vai ser transferida e adicionada a uma nova pasta. A nova pasta vai ser nomeada de acordo com o nome do repositório. A pasta vai conter o histórico completo do repositório remoto e a ramificação principal recém-criada.

2.4 - Salvando alterações no repositório:

Agora que você possui um repositório clonado ou inicializado, é possível fazer commit em alterações de versão do arquivo nele. O exemplo seguinte exemplifica criando e usando um arquivo de teste. Os passos a serem realizados neste exemplo são:

$ echo Conteúdo de teste para o tutorial de Git >> TesteCommit.txt
$ git add TesteCommit.txt ou git add .
$ git commit -m "Adicionado CommitTest.txt ao repositório"

Sobre os comandos usados: O comando "git add" é o que vai garantir que seus arquivos novos e/ou modificados sejam rastreados no seu próximo commit, assim sendo enviados do seu repositório local para o repositório remoto. O comando "git commit -m" define a mensagem que vai ser usada no seu commit, mensagem essa que recomenda-se ser uma breve explicação da(s) modificação(ões) feita(s).

2.5 - Enviando seu repositório local para o remoto:

Todas alterações que você fez até aqui, por enquanto, estão somente no seu repositório local. Para que elas sejam enviadas para um repositório remoto, é necessário usar o comando "git push". Tal comando é geralmente usado de duas maneiras: Se você clonou um repositório, tudo que precisa fazer é usar

$ git push

Agora, se você criou um repositório, precisa adicionar um "caminho" para o repositório remoto da seguinte forma:

$ git remote add nome link_do_repositório_remoto

Feito isso, basta usar o push da seguinte maneira:

$ git push nome main

Dessa forma, você enviará suas mudanças para o diretório especificado e já na branch principal (main).

3 - GitHub e GitPages

3.1 - O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. Ele permite que programadores, utilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo. GitHub é amplamente utilizado por programadores para divulgação de seus trabalhos ou para que outros programadores contribuam com o projeto, além de promover fácil comunicação através de recursos que relatam problemas ou mesclam repositórios remotos.

3.2 - Criando repositórios pelo GitHub

Ao acessar sua conta, você terá uma opção de criar um novo repositório no canto superior direito (um ícone de '+' ao lado da sua imagem de perfil).

Depois, clique na opção "new repository" que irá te redirecionar para outra página, na qual você deverá definir um nome para o seu repositório e, opcionalmente, uma descrição também. É recomendado que você deixe seu repositório público, mas você também pode deixá-lo privado se desejar.

Ao clicar no botão "create repository", você será direcionado à página do seu novo repositório com as instruções para envio do seu projeto local para o GitHub.

3.3 - Ativando o Pages

Para finalizar, vamos solicitar que o GitHub disponibilize um link do GitHub Pages para esse repositório. Para isso, você precisa ir até a aba "settings" do seu repositório.
